# Экраны MVP производственного контура

Дата создания: 17.06.2026

## Назначение документа

Этот документ описывает первый набор экранов MVP.

Цель - понять, какие рабочие интерфейсы нужны для запуска производственного контура, что видит каждый пользователь, какие действия он выполняет и какие данные попадают в ядро системы.

Главный принцип интерфейсов: минимум лишних действий. Пользователь должен делать только то, что реально нужно для работы, а система сама фиксирует время, состояние и историю.

## Общий принцип входа

Приложение одно для всех ролей.

Пользователь вводит свой доступ, после чего система определяет:

- сотрудника;
- роль;
- доступный интерфейс;
- нужно ли пройти первичную регистрацию;
- есть ли активная смена;
- есть ли активное назначение;
- есть ли незавершенный рейс или другое состояние.

## Экран 1. Единый вход

### Для кого

Для всех пользователей системы.

### Цель экрана

Открыть нужный интерфейс по роли сотрудника.

### Основные элементы

- поле ввода пинкода или пароля;
- кнопка `Войти`;
- сообщение об ошибке доступа;
- служебная информация о версии приложения.

### Что проверяет система

- существует ли доступ;
- активен ли доступ;
- к какому сотруднику он привязан;
- какая роль у доступа;
- какой интерфейс нужно открыть;
- требуется ли первичная регистрация.

### Что происходит после входа

- водитель открывает интерфейс водителя;
- машинист экскаватора открывает интерфейс экскаватора;
- горный мастер открывает интерфейс расстановки;
- диспетчер открывает диспетчерский экран;
- администратор открывает админку;
- руководство открывает отчетную витрину, если такая роль включена.

## Экран 2. Первичная регистрация водителя

### Для кого

Для водителя самосвала при первом входе.

### Цель экрана

Позволить водителю самостоятельно указать стартовые данные, чтобы администратор не вводил вручную всю первичную информацию.

### Основные поля

- смена: дневная или ночная;
- самосвал;
- общежитие;
- блок;
- секция.

### Основные действия

- выбрать смену;
- выбрать самосвал;
- выбрать общежитие;
- выбрать блок и секцию;
- нажать `Сохранить`.

### Что проверяет система

- сотрудник имеет роль водителя самосвала;
- выбранный самосвал свободен в этой смене;
- не превышены ограничения по самосвалу в вахте;
- общежитие доступно для заселения;
- не превышена вместимость секции;
- все обязательные поля заполнены.

### Что сохраняется в ядро

- сотрудник;
- роль;
- смена;
- самосвал;
- место проживания;
- дата и время регистрации;
- статус регистрации.

## Экран 3. Интерфейс водителя самосвала

### Для кого

Для водителя самосвала.

### Цель экрана

Дать водителю простой рабочий экран без лишних действий: открыть смену, видеть назначение, выполнить рейс, закрыть смену.

### Основные состояния экрана

1. Смена не открыта.
2. Смена открыта, назначение ожидается.
3. Есть новое назначение, нужно нажать `Принял`.
4. Есть активный рейс.
5. Рейс выполнен.
6. Смена готова к закрытию.

### Блоки экрана

- ФИО водителя;
- текущая смена;
- текущий самосвал;
- назначенный экскаватор;
- активная точка разгрузки;
- статус рейса;
- счетчик рейсов за смену;
- топливо, пробег, моточасы;
- уведомления.

### Основные действия

- `Открыть смену`;
- подтвердить или скорректировать топливо, пробег и моточасы;
- `Принял` для нового назначения;
- `Выполнено` после разгрузки;
- `Закрыть смену`.

### Правила интерфейса

- если новое назначение не принято, самосвал подсвечивается как проблема принятия задания;
- для нового назначения используется желтая визуальная индикация;
- если самосвал пришел из прошлой смены груженым, водитель сразу видит активный рейс;
- водитель не выбирает сам себе новую технику после первичной регистрации;
- водитель не должен вводить лишние комментарии без необходимости.

### Что сохраняется в ядро

- открытие смены;
- показатели на начало смены;
- подтверждение назначения;
- завершение рейса;
- показатели на конец смены;
- закрытие смены;
- все корректировки показателей.

## Экран 4. Интерфейс машиниста экскаватора

### Для кого

Для машиниста экскаватора.

### Цель экрана

Дать машинисту экскаватора возможность фиксировать фактическую работу экскаватора и создавать рейсы для назначенных самосвалов.

### Основные блоки экрана

- ФИО машиниста;
- текущая смена;
- экскаватор;
- горизонт;
- блок;
- порода;
- доступные точки разгрузки;
- назначенные самосвалы;
- статус экскаватора;
- простои;
- счетчик отправленных рейсов.

### Основные действия

- открыть смену;
- подтвердить экскаватор;
- указать горизонт;
- указать блок;
- выбрать породу;
- выбрать доступные точки разгрузки;
- отправить самосвал в рейс на точку разгрузки;
- изменить статус экскаватора;
- включить простой;
- завершить простой;
- закрыть смену.

### Механика создания рейса

Рабочая идея интерфейса:

- машинист видит иконки назначенных самосвалов;
- машинист выбирает самосвал;
- машинист выбирает точку разгрузки;
- система создает рейс;
- самосвал у машиниста становится неактивным до завершения рейса водителем;
- у водителя появляется активный рейс.

В будущей версии это можно сделать через перетаскивание иконки самосвала на точку разгрузки.

### Что сохраняется в ядро

- параметры работы экскаватора;
- создание рейса;
- смена загрузки;
- самосвал;
- точка разгрузки;
- порода;
- простои;
- статусы экскаватора;
- закрытие смены.

## Экран 5. Интерфейс горного мастера

### Для кого

Для горного мастера.

### Цель экрана

Сделать расстановку самосвалов по экскаваторам и контролировать ситуацию по комплексам.

### Основные блоки экрана

- список экскаваторов;
- комплексы экскаваторов;
- список доступных самосвалов;
- назначенные самосвалы под каждым экскаватором;
- неподтвержденные назначения;
- критические простои;
- предупреждения по технике;
- сдвинутые графики комплексов.

### Основные действия

- назначить самосвал под экскаватор;
- переназначить самосвал;
- снять самосвал с экскаватора;
- направить самосвал в ожидание/ОФР/ТО, если такой статус будет включен;
- видеть подтверждение `Принял` от водителя;
- видеть простои экскаватора;
- видеть проблему по комплексу.

### Что сохраняется в ядро

- назначение техники;
- изменение назначения;
- кто сделал назначение;
- когда сделано назначение;
- статус подтверждения;
- история расстановки.

## Экран 6. Диспетчерский экран

### Для кого

Для диспетчера.

### Цель экрана

Дать диспетчеру контроль смены и подготовку отчетов без ручной сборки данных из разных таблиц.

### Основные блоки экрана

- текущая дата;
- текущая смена;
- журнал последних диспетчерских действий;
- причины последних диспетчерских вмешательств;
- открытые смены водителей;
- открытые смены машинистов экскаваторов;
- активные экскаваторы;
- активные самосвалы;
- активные рейсы;
- закрытые рейсы;
- переходящие груженые рейсы;
- простои;
- неподтвержденные назначения;
- принятые назначения в работе;
- незакрытые смены;
- предупреждения для отчета;
- объем за смену;
- объем за сутки.

### Основные действия

- открыть контроль смены;
- проверить активные рейсы;
- проверить незакрытые смены;
- проверить принятые назначения в работе;
- закрыть смену сотрудника служебно;
- снять неактуальное назначение служебно;
- отменить ошибочный активный рейс;
- завершить активный рейс служебно;
- закрыть смену водителя или машиниста экскаватора служебно;
- внести корректировку в рамках прав;
- открыть отчетную витрину;
- выгрузить отчет в Excel по готовому шаблону.

### Что сохраняется в ядро

- служебное закрытие смены;
- корректировки;
- действия диспетчера;
- факты выгрузки отчетов;
- проверка отчетных данных.

## Экран 7. Механическая служба

### Для кого

Для механиков и ответственных сотрудников механической службы.

### Цель экрана

Вынести механические простои из общих комментариев смены в отдельный управляемый процесс: увидеть технику, по которой уже есть сигнал о простое, открыть механическое событие, выбрать причину и закрыть простой после устранения проблемы.

### Основные блоки экрана

- техника, где уже виден простой в активной смене;
- открытые механические простои;
- причина простоя из справочника;
- комментарий механика;
- время открытия простоя;
- длительность открытого простоя;
- ссылка на диспетчерский пульт.

### Основные действия

- открыть механический простой по технике;
- выбрать причину простоя;
- добавить короткий комментарий;
- закрыть механический простой;
- перейти к диспетчерскому пульту для сверки сменной ситуации.

### Правила интерфейса

- машинист экскаватора или диспетчер фиксирует факт простоя на своем уровне;
- механик уточняет именно механическую причину, не перегружая машиниста деталями ремонта;
- справочник причин простоя управляется через админку;
- в рабочем экране механика показываются только активные и нужные для смены события.

### Что сохраняется в ядро

- техника;
- причина простоя;
- время открытия;
- время закрытия;
- комментарий;
- сотрудник, который открыл или закрыл событие;
- связь с общей картиной смены и будущей отчетностью по простоям.

## Экран 8. Админка

### Для кого

Для администратора системы.

### Цель экрана

Управлять пользователями, ролями, справочниками, первичными настройками и шаблонами отчетов.

### Основные разделы

- сотрудники;
- роли;
- доступы;
- техника;
- виды техники;
- модели техники;
- породы;
- коэффициенты;
- точки разгрузки;
- причины простоев;
- общежития;
- блоки и секции;
- смены и вахты;
- комплексы экскаваторов;
- шаблоны отчетов;
- журнал действий.

### Основные действия

- создать сотрудника;
- выдать доступ;
- назначить роль;
- отключить доступ;
- добавить технику;
- изменить справочник;
- создать шаблон отчета;
- выбрать столбцы отчета;
- сохранить шаблон;
- просмотреть журнал действий.

### Что сохраняется в ядро

- сотрудники;
- роли;
- права;
- справочники;
- шаблоны отчетов;
- история изменений.

## Экран 9. Отчеты и шаблоны

### Для кого

Для администратора и диспетчера.

### Цель экрана

Позволить формировать отчеты первой очереди и выгружать их в Excel.

### Права в MVP

- администратор создает и меняет шаблоны отчетов;
- диспетчер использует готовые шаблоны;
- диспетчер выгружает отчеты в Excel;
- позже права на создание шаблонов можно расширить.

### Основные блоки экрана

- список шаблонов;
- тип отчета;
- период;
- смена;
- выбранные столбцы;
- предпросмотр отчета;
- предупреждения по данным;
- кнопка выгрузки в Excel.

### Основные действия администратора

- создать шаблон;
- выбрать тип отчета;
- выбрать доступные столбцы;
- изменить порядок столбцов;
- сохранить шаблон под названием;
- изменить шаблон.

### Основные действия диспетчера

- выбрать готовый шаблон;
- выбрать дату или период;
- проверить предупреждения;
- сформировать отчет;
- выгрузить отчет в Excel.

### Отчет по механическим простоям

Отдельный отчет первой версии для диспетчера, механика, администратора и руководства.

Основные возможности:

- фильтр по дате начала простоя;
- фильтр по статусу: все, открытые, закрытые;
- фильтр по технике;
- фильтр по причине простоя;
- показатели: всего событий, открытые, закрытые, критические, суммарная длительность;
- таблица событий простоев;
- выгрузка в Excel.

### Что сохраняется в ядро

- шаблон отчета;
- настройки столбцов;
- кто создал шаблон;
- кто изменил шаблон;
- факт формирования отчета;
- факт выгрузки отчета.

## Экран 10. Витрина руководства

### Для кого

Для руководства компании.

### Цель экрана

Дать быструю картину по производству без погружения в диспетчерские таблицы.

### Блоки первой версии

- объем за текущую смену;
- объем за текущие сутки;
- объем по экскаваторам;
- объем по направлениям;
- количество рейсов;
- техника в работе;
- техника в простое;
- главные отклонения.

### Обновление 18.06.2026

В первой рабочей версии витрины добавлен блок `День против ночи`.

Он показывает по выбранной дате:

- факт по дневной и ночной смене;
- план по дневной и ночной смене;
- отклонение факт минус план;
- количество рейсов;
- визуальные полосы факта и плана.

Если рейс не связан со сменой загрузки или разгрузки, он попадает в строку `Смена не указана`.

### Добавлено 18.06.2026

Витрина руководства получила блок `Динамика за 7 дней`.

Что показывает блок:

- факт по объему за каждый день;
- план по объему за каждый день;
- отклонение факта от плана;
- процент выполнения плана;
- количество завершенных рейсов;
- визуальные полосы факта и плана;
- выделение выбранной даты.

Назначение блока: быстро показать руководству не только состояние выбранных суток, но и направление движения за неделю - рост, просадку или стабильное выполнение плана.

Также в верхние KPI витрины добавлен показатель `Выполнение плана` за выбранные сутки.

Недельная динамика дополнительно получила итоговую сводку:

- итоговый факт за 7 дней;
- план за 7 дней;
- отклонение за неделю;
- процент выполнения недельного плана;
- количество рейсов за неделю;
- лучший день по объему;
- день с самой сильной просадкой.

### Правило MVP

Витрина руководства не должна задерживать запуск производственного учета.

Если времени не хватает, она делается после базовых экранов водителя, машиниста экскаватора, горного мастера, диспетчера, админки и отчетов.

## Общая последовательность экранов для запуска

1. Единый вход.
2. Карта интерфейсов MVP.
3. Админка: сотрудники, роли, доступы.
4. Админка: справочники.
5. Первичная регистрация водителя.
6. Интерфейс водителя.
7. Интерфейс машиниста экскаватора.
8. Интерфейс горного мастера.
9. Диспетчерский экран.
10. Механическая служба.
11. Отчеты и шаблоны.
12. Витрина руководства.

## Экран 11. Карта интерфейсов MVP

### Для кого

Для администратора, разработчика, руководителя при демонстрации и всех, кто проверяет текущий MVP.

### Цель экрана

Собрать все готовые веб-интерфейсы в одном месте, чтобы не искать ссылки в чате и не держать карту системы в голове.

### Блоки первой версии

- вход и администрирование;
- рабочие интерфейсы;
- отчеты и руководство;
- демо-коды доступа;
- краткие пояснения, для чего нужен каждый интерфейс.

### Правило MVP

Карта интерфейсов не заменяет права доступа и полноценное меню системы. Это служебная навигационная страница для быстрой проверки текущей сборки.

## Критерии готовности первого набора экранов

Первый набор экранов можно считать готовым, если:

- пользователь входит через единый вход;
- пользователь может открыть карту готовых интерфейсов MVP;
- система открывает интерфейс по роли;
- администратор создает сотрудника и доступ;
- водитель проходит первичную регистрацию;
- водитель открывает смену;
- водитель подтверждает топливо, пробег и моточасы;
- горный мастер назначает самосвал под экскаватор;
- водитель подтверждает назначение кнопкой `Принял`;
- машинист экскаватора создает рейс;
- водитель завершает рейс кнопкой `Выполнено`;
- диспетчер видит смену, рейсы, объемы и предупреждения;
- механик видит технику с механическим простоем и может открыть/закрыть событие простоя;
- администратор создает шаблон отчета;
- диспетчер выгружает отчет в Excel.

## Текущий вывод

Первый набор экранов должен обслуживать главный производственный цикл:

сотрудник вошел -> открыл смену -> техника назначена -> рейс создан -> рейс выполнен -> объем попал в отчет.

Если этот цикл работает, система уже начинает решать главную боль диспетчерской.

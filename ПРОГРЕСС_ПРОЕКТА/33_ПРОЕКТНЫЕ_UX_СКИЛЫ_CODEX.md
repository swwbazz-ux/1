# Проектные UX-скилы Codex

Дата создания: 20.06.2026

## Зачем добавлены

Для дальнейшей работы над интерфейсами рабочих мест подготовлены отдельные проектные скилы Codex.

Цель: чтобы Codex при доработке интерфейсов не ограничивался технической правкой шаблонов, а стабильно проверял экран как рабочий инструмент конкретной роли: водителя, машиниста экскаватора, горного мастера, диспетчера, администратора или руководства.

## Созданные скилы

### `accounting-system-ux-review`

Путь:

`ПРОЕКТНЫЕ_СКИЛЫ/accounting-system-ux-review`

Назначение:

- проверять экран через роль пользователя и реальный сменный сценарий;
- отделять основное действие от вторичных проверок;
- снижать количество лишних действий;
- учитывать мобильный, планшетный и desktop-сценарии;
- выявлять UX-риски до и после реализации.

### `accounting-system-design-system`

Путь:

`ПРОЕКТНЫЕ_СКИЛЫ/accounting-system-design-system`

Назначение:

- удерживать единый визуальный язык проекта;
- не допускать случайных разных стилей на разных экранах;
- переиспользовать существующие карточки, таблицы, формы, фильтры, шапки и статусы;
- согласовывать Figma и код;
- поддерживать интерфейс как рабочую систему, а не как набор разрозненных страниц.

### `accounting-system-browser-qa`

Путь:

`ПРОЕКТНЫЕ_СКИЛЫ/accounting-system-browser-qa`

Назначение:

- после UI-изменений проверять реальный экран в браузере;
- запускать сервер только стандартным способом через `START_SERVER_MVP.bat`;
- проверять desktop и mobile viewports;
- смотреть переполнения, наложения текста, видимость кнопок, формы, таблицы, карточки и консольные ошибки;
- фиксировать в ответе только фактически проверенные результаты.

## Проверка структуры

Проверено:

- у каждого скила есть `SKILL.md`;
- у каждого скила есть `agents/openai.yaml`;
- во всех `SKILL.md` есть обязательные поля `name` и `description`;
- базовая структура прошла локальную проверку без установки дополнительных пакетов.

Стандартный валидатор `skill-creator/scripts/quick_validate.py` в текущей среде не запустился, потому что в доступных Python-окружениях отсутствует модуль `yaml`. Установка дополнительных пакетов для этой проверки не выполнялась.

## Ограничение установки

Автоматическая установка в глобальную папку Codex:

`C:\Users\swwba\.codex\skills`

не выполнена, потому что текущая сессия имеет доступ на запись только в рабочую папку проекта.

Скилы подготовлены в проектной папке:

`ПРОЕКТНЫЕ_СКИЛЫ`

Их можно использовать как проектные инструкции или позже перенести в глобальную папку Codex для автоподключения.

## Как использовать в следующих чатах

В следующем рабочем чате можно дать Codex инструкцию:

> Перед интерфейсными задачами используй проектные скилы из папки `ПРОЕКТНЫЕ_СКИЛЫ`: `accounting-system-ux-review`, `accounting-system-design-system`, `accounting-system-browser-qa`.

Для глобального подключения нужно перенести папки скилов в:

`C:\Users\swwba\.codex\skills`

После этого Codex должен видеть их как обычные пользовательские скилы.

# Состояние диспетчерского модуля на 25.06.2026

Документ создан как точка восстановления и передачи работы по модулю Горного диспетчера после завершения основного этапа дизайна, подключения к текущему ядру Django и очистки от клиентской имитации.

## Главный экран

Основной рабочий экран: `/dispatcher/control/`.

Статус: визуальная структура пульта принята пользователем как базовая. Рабочую зону пульта не менять без отдельного решения. Допустимы только исправления механики, привязки к данным, ошибок и аккуратные изменения общей шапки, если они не уменьшают рабочую зону.

Рабочая логика экрана:

- слева гараж экскаваторов;
- по центру активные комплексы;
- справа гараж самосвалов;
- комплекс формируется активным экскаватором;
- номер комплекса соответствует номеру экскаватора;
- самосвал физически находится только в одном месте: гараж или один комплекс;
- пустой комплекс без экскаватора не принимает самосвалы;
- перенос комплекса в гараж самосвалов сбрасывает самосвалы, но оставляет экскаватор активным;
- перенос комплекса в гараж экскаваторов сбрасывает самосвалы и возвращает экскаватор в гараж.

## Данные и границы имитации

Из рабочего пульта убраны демо-подстановки объемов и техники. Экран должен показывать только то, что есть в текущей базе.

Текущие правила данных:

- активные комплексы берутся из `ExcavatorPlacement` и активных назначений `HaulAssignment`;
- сам активный рейс `Trip` больше не создает комплекс автоматически;
- сам активный рейс `Trip` больше не убирает самосвал из гаража;
- гараж самосвалов строится из реальных `Equipment`;
- самосвал 53 исключен из гаража самосвалов как эвакуатор;
- pending-назначение самосвала подсвечивает сам самосвал, но не красит весь комплекс в желтый;
- комплекс становится красным при открытом простое экскаватора;
- если нет реальных рейсов и назначений, сменные отчеты в карточках остаются пустыми, а не заполняются демо-строками.

Оставшаяся допустимая заглушка: процент выполнения сменного плана может быть `0`, пока в системе нет полноценной модели сменного плана по технике/комплексу.

## Серверная механика DnD

DnD пульта переведен на серверные POST-обработчики:

- `dispatcher_move_excavator_view`
- `dispatcher_assign_truck_view`

Маршруты:

- `/dispatcher/control/excavator/move/`
- `/dispatcher/control/truck/assign/`

После успешного POST пульт перезагружается, чтобы DOM всегда соответствовал базе. Оптимистичная клиентская перерисовка намеренно не используется.

Важно: старые клиентские функции, которые сами создавали комплексы, назначали самосвалы или возвращали технику в гараж без записи в БД, удалены из `dispatcher_control.html`. Также удален CSS-компонент `.dispatcher-drop-chip`, который рисовал синие фантомные подсказки вида `1 -> K-*`.

## Обновление 25.06.2026: правила открытия и закрытия смены

Закрытая смена Горного диспетчера блокирует рабочие изменения на `/dispatcher/control/`: DnD, перенос экскаваторов, назначение/снятие самосвалов, сброс комплекса и служебные корректировки доступны только при открытой смене Горного диспетчера.

При начале новой смены сбрасываются только сменные показатели. Расстановка техники, активные комплексы, состояния техники и текущие назначения не очищаются. Показатели смены считаются от `opened_at` текущей смены, поэтому новая смена начинается с нулевого среза без разрушения производственной расстановки.

## Как поднять и проверить после перерыва

Запускать и останавливать сервер только штатными файлами проекта:

```powershell
cmd /c "echo. | STOP_SERVER_MVP.bat"
cmd /c "echo. | START_SERVER_MVP.bat"
```

Если `START_SERVER_MVP.bat` зависает в терминале на `pause`, это нормально для текущего bat-файла. Нужно проверить порт и страницу:

```powershell
Test-NetConnection 127.0.0.1 -Port 8000
```

Основная ручная проверка:

1. Открыть `http://127.0.0.1:8000/dispatcher/control/`.
2. Перетащить экскаватор из гаража в активную зону.
3. Перетащить самосвал в существующий комплекс.
4. Перетащить самосвал из комплекса в гараж самосвалов.
5. Перетащить комплекс в гараж самосвалов: самосвалы должны сброситься, экскаватор остаться активным.
6. Перетащить комплекс в гараж экскаваторов: комплекс должен расформироваться, самосвалы вернуться в гараж, экскаватор вернуться в гараж.
7. Перетащить самосвал в пустой комплекс без экскаватора: ничего не должно происходить, предупреждение не показывать.

## Известные ограничения

- Для действий назначения сейчас используется `DispatcherActionType.CANCEL_ASSIGNMENT`, потому что в модели журнала пока нет отдельного типа действия для изменения назначения. При развитии журнала стоит добавить отдельные типы: `ASSIGN_TRUCK`, `RELEASE_TRUCK`, `MOVE_EXCAVATOR`, `DISBAND_COMPLEX`.
- Проценты плана техники требуют будущей модели сменного плана.
- Часть отчетных экранов уже связана с текущими моделями, но полноценная промышленная аналитика потребует уточнения справочников планов, горизонтов, блоков, типов пород и точек разгрузки.

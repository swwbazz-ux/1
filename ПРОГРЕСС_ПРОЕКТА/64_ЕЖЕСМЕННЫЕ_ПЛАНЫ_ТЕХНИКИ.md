# Ежесменные планы техники

Дата фиксации: 09.07.2026.

## Решение

Основным способом задания плана техники становится ежесменный план группы техники.

Администратор больше не должен ежедневно создавать план на конкретную дату, смену `day/night` и каждую единицу техники. Старые сущности `ShiftPlan` и `EquipmentShiftPlan` остаются для истории, совместимости отчетов и резервного расчета, но рабочий контур новых смен должен идти через группу техники и snapshot открытой смены сотрудника.

## Новая сущность

Добавлена сущность `EquipmentPlanGroup` - группа плана техники.

Рабочие группы первого контура:

- `Самосвалы БелАЗ` - расчет по рейсам;
- `Самосвалы NHL` - расчет по рейсам;
- `Экскаваторы 4000` - расчет по м3;
- `Экскаваторы 3000` - расчет по м3.

Группа хранит:

- название и код;
- тип расчета: рейсы или м3;
- значение ежесменного плана;
- список техники в группе;
- активность;
- дату начала действия;
- комментарий;
- кто изменил;
- когда изменил.

Стартовая миграция создает эти группы и раскладывает уже существующую технику как начальную настройку. В самом расчете принадлежность техники не хардкодится: сервер ищет активную группу через связь `EquipmentPlanGroup.equipment`.

## Snapshot смены

При открытии смены сотрудника сервер:

1. определяет технику сотрудника;
2. ищет группу плана этой техники;
3. проверяет активность группы и наличие значения плана;
4. записывает snapshot в `EmployeeShift`.

В `EmployeeShift` фиксируются:

- `plan_group`;
- `plan_group_name`;
- `plan_calculation_mode`;
- `plan_value`;
- `plan_assigned_at`;
- `plan_status`.

Если администратор изменит группу позже, старые открытые и закрытые смены не пересчитываются задним числом: расчет прогресса использует snapshot смены.

## Статусы отсутствия плана

Отсутствие плана больше не маскируется как `0%`.

Сервер возвращает отдельные статусы:

- `no_plan_group` - техника не включена ни в одну группу плана;
- `no_active_plan` - группа есть, но она неактивна или значение плана не задано.

Интерфейсы Водителя и Машиниста экскаватора показывают эти состояния отдельной подписью, а не обычным нулевым прогрессом.

## Расчет выполнения

Для самосвала процент считается по snapshot-плану его текущей смены:

`выполненные рейсы / план рейсов * 100`.

Для экскаватора процент считается по snapshot-плану его текущей смены:

`отгруженный объем м3 / план м3 * 100`.

Экран `/excavator/work/` для карточек самосвалов берет snapshot текущей смены самосвала, а не план экскаватора и не старый дневной `EquipmentShiftPlan`.

## Админка

В рабочей админке MVP добавлен раздел `Ежесменные планы техники`:

`/system-admin/references/equipment-plan-groups/`.

Старые разделы `Сменные планы` и `Планы техники` остаются в админке как исторические/совместимые, но новый ежедневный рабочий процесс должен начинаться с раздела `Ежесменные планы техники`.

## Дополнение 09.07.2026: пульт диспетчера подключен к snapshot-планам смен

Экран `/dispatcher/control/` больше не использует старую дневную схему для статусбаров техники. Пульт берет данные выполнения из текущей открытой `EmployeeShift`: `plan_status`, `plan_group`, `plan_group_name`, `plan_calculation_mode`, `plan_value` и snapshot, который был назначен при старте смены.

Для самосвалов процент считается как `выполненные рейсы / snapshot plan_value * 100`, для экскаваторов - как `отгруженный м3 / snapshot plan_value * 100`. Статусы `no_plan_group`, `no_active_plan` и отсутствие snapshot отображаются отдельными подписями, а не превращаются в фальшивые `0%`.

В серверный контекст и JSON-карточки пульта добавлены поля плана: статус, название группы, режим расчета, значение плана, факт, процент и единица измерения. В шаблоне появились видимые строки `процент`, `факт / план`, группа плана и отдельное состояние отсутствия плана для самосвалов БелАЗ/NHL и экскаваторов 4000/3000. PWA-оболочка диспетчера поднята до `dispatcher-desktop-shell-v23`.

Проверки локально: `manage.py check` - OK, полный `manage.py test` - 209 тестов OK. Добавлены тесты пульта на snapshot-план самосвала, snapshot-план экскаватора, расчет рейсов/м3, статусы `no_plan_group` / `no_active_plan` без `0%`, а также неизменность прогресса открытой смены после изменения группового плана.

# Исправление 09.07.2026: защита группировки техники в разделе "Ежесменные планы техники"

- Админка `/system-admin/references/equipment-plan-groups/` редактирует поле "Техника в группе" через прокручиваемые чекбоксы, а не через ежедневный ручной план и не через нестабильный multiselect.

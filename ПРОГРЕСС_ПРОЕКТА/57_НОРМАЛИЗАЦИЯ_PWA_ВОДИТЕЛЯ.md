# Нормализация PWA водителя самосвала

Дата: 2026-07-07

## Решение

Экран `Работа` PWA водителя очищен от конфликтующих поколений CSS и переведен на один компонент `WorkTab`.

Удалены старые рабочие слои:

- `driver-shell.is-work-tab`;
- `driver-ref-*`;
- `driver-status-ring`;
- фиксированный stage `941 x 1672`;
- `transform scale`;
- ветка `Новое назначение` / `Принять`.

Водитель не принимает назначение вручную на главном экране. Он видит назначенный экскаватор и точку разгрузки.

## Геометрия

Базовая геометрия водительского PWA:

- `body.driver-mobile-screen`: `100dvh`, `overflow: hidden`;
- `.driver-shell`: `width: min(100vw, 430px)`, `height: 100dvh`;
- сетка экрана: `header / content / bottom-nav`;
- нижняя навигация единая для вкладок `Работа`, `Смена`, `Простои`, `Путевка`.

## Главная кнопка

Круговая кнопка построена через SVG:

- внешний серый `track`;
- внешний зеленый `progress` через `stroke-dasharray` и `stroke-dashoffset`;
- отдельный `hold-progress` для удержания 2 секунды;
- внутренний круг-кнопка поверх SVG;
- контейнер имеет `aspect-ratio: 1 / 1`.

Разрешенные состояния:

- `empty`: `ПУСТОЙ`;
- `loaded`: `ЗАГРУЖЕН`, удержание 2 секунды отправляет `trip_unloaded`;
- `pending`: `ОТПРАВКА`;
- `error`: `ОШИБКА`.

## Проверка

Проверено локально и на `driverform.ru`:

- `manage.py check`;
- профильные тесты водителя;
- браузер 390 x 844;
- круг, SVG и кнопка имеют одинаковую ширину и высоту;
- нижняя навигация не перекрывает контент;
- в DOM нет `Принять`, `Новое назначение`, `ПРИНЯТЬ`;
- версия PWA: `driver-mobile-shell-v32`.

## Что представляет собой старая форма

Старая форма построена как широкая почасовая матрица.

Основные признаки:

- отдельные листы по датам и сменам, например `01.03 см1`, `01.03 см2`;
- строки сгруппированы по экскаваторам;
- внутри блока экскаватора есть строки по типам самосвалов, например `БелАЗ`, `NHL`;
- по каждому часу есть набор колонок по грузу и направлению;
- встречаются категории `рыхлая`, `скала`, `ККД`, `СКДР`, `склад`, `НЕГАБ`;
- итоговые строки показывают `РЕЙСЫ` и `ОБЪЕМЫ`.

## Как это закрывается в MVP

В первом MVP не создается отдельная точная копия старой 128-колоночной матрицы.

Вместо этого в отчет по объемам добавлена группировка:

`Час выполнения рейса`

Рабочая ссылка:

`/reports/volume/?group_by=completed_hour`

Что показывает группировка:

- час выполнения рейса;
- общий объем за этот час;
- общий тоннаж за этот час;
- количество рейсов за этот час.

Excel-выгрузка отчета по объемам поддерживает эту же группировку.

## Почему так

Старая почасовая форма слишком широкая и жестко привязана к Excel-логике.

Для первого пилота важнее проверить главный поток данных:

1. рейс создается в системе;
2. у рейса есть время выполнения;
3. система умеет собрать рейсы по часам;
4. диспетчер может выгрузить почасовую сводку в Excel.

Если диспетчерской после пилота понадобится именно старая матрица, ее можно сделать отдельной формой на основе тех же данных.

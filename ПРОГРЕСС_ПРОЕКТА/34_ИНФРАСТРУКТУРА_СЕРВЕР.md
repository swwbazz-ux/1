# Инфраструктура и сервер для производственного контура

## Оценка нагрузки

Исходные ориентиры:

- до 60 самосвалов;
- около 10 экскаваторов;
- 2 смены;
- примерно 20 рейсов на самосвал за смену;
- около 500 сотрудников;
- оперативное хранение производственных данных минимум за 3 месяца.

Расчетная базовая нагрузка по рейсам:

- 60 самосвалов x 20 рейсов x 2 смены = около 2400 рейсов в сутки;
- 90 дней = около 216000 рейсов;
- с событиями, простоями, изменениями статусов, журналами и отчетными срезами база может быстро выйти на сотни тысяч или несколько миллионов строк, что нормально для PostgreSQL при правильных индексах.

## Минимальная и рекомендуемая конфигурация

Для реального производственного запуска не использовать SQLite и обычный виртуальный хостинг. Нужен PostgreSQL.

Минимум для пилота:

- 4-6 vCPU;
- 16 ГБ RAM;
- 200-300 ГБ NVMe;
- ежедневные резервные копии;
- мониторинг диска, CPU, RAM и PostgreSQL.

Комфортная конфигурация для спокойной работы:

- 8 vCPU или выделенный CPU сопоставимого класса;
- 32 ГБ RAM;
- 500 ГБ-1 ТБ NVMe;
- отдельное место под резервные копии;
- PostgreSQL, приложение и фоновые задачи можно начать на одном сервере, но с возможностью дальнейшего разделения.

## Ориентир по SprintHost / Sprintbox на 21.06.2026

Публичные тарифы Sprintbox можно использовать для теста и пилота, но они выглядят тесными для спокойного промышленного запуска.

Подходящие из VDS только условно:

- `Про 12`: 12 ядер, 16 ГБ RAM, 120 ГБ NVMe, 10700 руб/мес - минимально допустимо для пилота, но мало диска и RAM без запаса;
- `AMS x16`: 10 ядер, 16 ГБ RAM, 200 ГБ NVMe, 6999 руб/мес - лучше по диску, но зарубежная локация и 200 Мбит/с, для российской производственной эксплуатации спорно.

Предпочтительный вариант у SprintHost для реальной эксплуатации - выделенный сервер, а не VDS:

- целевой уровень: 32 ГБ RAM, 6-8+ физических ядер / 12-16 потоков, NVMe от 500 ГБ, лучше 2 x NVMe и отдельный бекапный диск;
- публичная страница выделенных серверов SprintHost показывает варианты с 2 x 1 ТБ SSD NVMe и ценами порядка 12500-18500 руб/мес в зависимости от акции/конфигурации.

## Практический вывод

Для проекта лучше планировать:

1. Разработка и демонстрация - локально или младший VDS.
2. Пилот - старший VDS уровня 10-12 vCPU / 16 ГБ RAM, если данных немного и есть ежедневные бекапы.
3. Производственный контур - выделенный сервер с 32 ГБ RAM и NVMe, PostgreSQL, регулярными бекапами и мониторингом.

## Дополнение 21.06.2026: сравнение российских провайдеров с оплатой в рублях

Критерии исследования:

- российский провайдер или российская площадка;
- возможность оплаты в рублях;
- конфигурация не слабее пилотного уровня: примерно 8 vCPU / 16 ГБ RAM / NVMe;
- для промышленного запуска предпочтительнее 32 ГБ RAM и выделенный сервер.

Предварительный рейтинг:

1. `HOSTKEY vm.v3-heavy` - 8 vCPU, 32 ГБ RAM, 240 ГБ NVMe, около 5050 руб/мес. Лучшее соотношение цена/ресурсы среди найденных VDS, но это все равно виртуальный сервер и нужно отдельно контролировать бекапы/лимит трафика.
2. `Timeweb Cloud High` - 8 CPU, 16 ГБ RAM, 160 ГБ NVMe, около 4200 руб/мес. Выгодно для пилота, но без запаса по RAM и диску.
3. `REG.RU High C8-M16-D180` - 8 vCPU, 16 ГБ RAM, 180 ГБ NVMe, около 8000 руб/мес. Дороже HOSTKEY/Timeweb, но понятная российская площадка и несколько регионов.
4. `Sprintbox Про 12` - 12 CPU, 16 ГБ RAM, 120 ГБ NVMe, около 10700 руб/мес. По CPU хорошо, но мало диска и RAM за свои деньги.
5. `REG.RU dedicated Ryzen 7 7700X` - 8 ядер / 16 потоков, 32 ГБ RAM, 2 x 1 ТБ NVMe, около 19100 руб/мес, при оплате на год около 16235 руб/мес. Практичный вариант для промышленного контура.
6. `Selectel AR43-NVMe` - около 24900 руб/мес. Дороже, но сильнее как корпоративный вариант.

Вывод:

- самый выгодный бюджетный старт: HOSTKEY VDS с 32 ГБ RAM;
- самый разумный промышленный вариант: физический сервер REG.RU или аналогичный dedicated с 32 ГБ RAM и 2 x NVMe;
- крупные облака Yandex Cloud, VK Cloud, Cloud.ru удобны корпоративно, но по цене для этой задачи обычно хуже VDS/dedicated.

## Дополнение 21.06.2026: сервер для демонстрации производственного контура

Для демонстрации только производственного контура с диспетчерской и архивом данных за 1 месяц выделенный сервер не нужен.

Оценка демонстрационного объема:

- 60 самосвалов x 20 рейсов x 2 смены x 30 дней = около 72000 рейсов;
- даже с событиями рейсов, простоями, журналами действий и служебными статусами это остается умеренной нагрузкой для PostgreSQL.

Минимально разумная конфигурация:

- 4 vCPU;
- 8 ГБ RAM;
- 120 ГБ NVMe;
- PostgreSQL на том же сервере;
- ежедневный бекап базы.

Нормальная конфигурация для спокойной демонстрации:

- 8 vCPU;
- 16 ГБ RAM;
- 160 ГБ NVMe;
- PostgreSQL, приложение и фоновые задачи на одном сервере;
- ежедневные бекапы и контроль свободного места.

Предпочтительный вариант из найденных тарифов:

- `HOSTKEY vm.v2-medium`: 8 vCPU, 16 ГБ RAM, 160 ГБ NVMe, около 3390 руб/мес - оптимально для демонстрации;
- `HOSTKEY vm.v2-heavy`: 8 vCPU, 32 ГБ RAM, 240 ГБ NVMe, около 4420 руб/мес - лучше, если разница в цене некритична и хочется запас.

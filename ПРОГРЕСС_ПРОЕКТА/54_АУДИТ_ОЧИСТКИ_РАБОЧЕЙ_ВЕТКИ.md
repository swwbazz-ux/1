# Аудит очистки рабочей ветки

Дата: 05.07.2026

## Причина

В рабочей ветке `codex/pilot-scenario-validation` накопились изменения из нескольких направлений: PWA водителя, экран экскаваторщика, справочник простоев, состояния техники, сменные планы, realtime, отчеты и проектные документы. Из-за этого панель Codex показывает большой общий дифф, который невозможно безопасно воспринимать как одну задачу.

## Что уже очищено

- Локальные остатки неудачной попытки `driver-mobile-shell-v28` убраны из рабочих файлов водителя.
- Локальные файлы `templates/users/driver_shift.html` и `users/views.py` синхронизированы с фактическим production-состоянием.
- Тестовые ожидания версии приведены к текущей связке: HTML `driver-mobile-shell-v27`, service worker `driver-mobile-shell-v21`.
- QA PNG-скриншоты `driver-v*`, `driver-work-v*`, `excavator-*`, `mining-master-*` перенесены в локальный архив `backups/local/qa-screenshots-*`.
- В `.gitignore` добавлены правила для локальных QA-артефактов и папки `backups/`.

## Что осталось в рабочей ветке

Оставшиеся изменения не считаются простым мусором до отдельной проверки. Их нужно разбирать пакетами:

1. Серверные справочники и состояния техники.
2. Серверный справочник причин простоев и цвета простоев.
3. Механика рейсов `truck_loaded` / `trip_unloaded`.
4. PWA водителя.
5. Экран экскаваторщика.
6. Сменные планы и сменная аналитика.
7. Realtime-клиент и тихая синхронизация.
8. Проектные документы `ПРОГРЕСС_ПРОЕКТА`.

## Правило дальнейшей работы

Нельзя продолжать править PWA водителя поверх текущей грязной ветки без предварительного разделения изменений. Следующий безопасный порядок:

1. Снять текущий `git status --short` и сгруппировать файлы по смысловым пакетам.
2. Для каждого пакета отдельно решить: оставить, оформить коммитом, перенести в отдельную ветку или откатить.
3. Не удалять миграции, модели, серверные справочники и тесты без понимания, опубликованы ли они уже на `driverform.ru`.
4. Перед новой правкой PWA водителя сначала зафиксировать чистую точку: актуальный production-скриншот, локальный baseline и список файлов, которые разрешено трогать.

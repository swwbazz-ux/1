# Маршрутизация рабочих чатов Codex

Дата: 2026-07-08.

Создан постоянный документ `CODEX_CHAT_ROUTING.md` для маршрутизации специализированных рабочих чатов Codex по проекту Copper Resources.

## Что зафиксировано

- Создан `CODEX_CHAT_ROUTING.md`.
- Заведены новые специализированные чаты:
  - `ШТАБ — Copper Resources`;
  - `Водитель — PWA`;
  - `Экскаваторщик — PWA`;
  - `Горный мастер — PWA`;
  - `Диспетчер — PWA`;
  - `Сервер и публикация`;
  - `Realtime и обмен данными`.
- Правила чатов перенесены из первого сообщения в документ проекта.
- Для каждого профиля указаны назначение, документы для чтения, разрешенные рабочие зоны и запреты.
- Добавлены общие правила: начинать с `git status`, останавливаться при грязном дереве, выполнять commit/push/deploy только по отдельному разрешению.
- Добавлен способ восстановления контекста в длинном чате через запрос профиля из `CODEX_CHAT_ROUTING.md`.
- Добавлены короткие стартовые сообщения для новых чатов.

## Что не менялось

- Код не менялся.
- UI не менялся.
- `templates`, `views`, `static`, `models`, `urls` не менялись.
- БД, `.env`, production, deploy, migrate, collectstatic не трогались.
- Сервер не запускался.

## Связанные правки

- В `AGENTS.md` добавлено правило: если пользователь указал профиль чата, Codex должен использовать `CODEX_CHAT_ROUTING.md` и соответствующий раздел как рабочую рамку.
- В `CODEX_WORKFLOW.md` добавлена короткая ссылка на `CODEX_CHAT_ROUTING.md` для специализированных рабочих чатов.

# Концепция мобильного модуля Горного мастера

Дата: 22.06.2026

## Кодовый перенос 22.06.2026

Первая рабочая версия модуля перенесена в Django на экран `/mining-master/assignments/`.

Перенесено в код:

- рабочая доска комплексов;
- плитки техники с силуэтом, крупным номером и цветом состояния;
- дневная и ночная тема;
- DnD-расстановка самосвалов по экскаваторам;
- снятие самосвала в гараж;
- режим наблюдения без открытой смены;
- начало и завершение смены через подтверждение;
- поиск и фильтры техники;
- стартовый дашборд смены;
- desktop-адаптация без боковой панели.

Граница MVP в коде: экран использует существующие модели `HaulAssignment`, `Trip`, `DowntimeEvent`, `EmployeeShift`, `Equipment`. Полноценные операции `ОФР`, `ремонт`, `резерв` пока не записываются отдельными действиями Горного мастера, потому что для этого нужна согласованная модель событий/статусов техники и связь с механиками/простоями.

Проверка переноса:

- `manage.py check` - ошибок нет;
- `manage.py test assignments` - 5 тестов пройдены;
- Browser QA через Chrome/Playwright: desktop и mobile `390px`, без горизонтального переполнения и ошибок консоли.

Документ фиксирует цельную концепцию будущего мобильного модуля Горного мастера перед отрисовкой Figma-макетов. Главная задача документа - убрать повторные уточнения в чате и держать единый источник решений по логике, экранам, состояниям и границам роли.

## Figma-макет

22.06.2026 создан Figma-файл концепции:

- состав: легенда логики, вариант A `Плотный рабочий`, вариант B `Крупные touch-зоны`;
- в каждом варианте собрано по 13 мобильных экранов;
- всего собрано 26 мобильных экранов;
- базовый формат экранов: смартфон 390 x 844, вертикальная ориентация;
- вариант A проверяет плотный рабочий интерфейс;
- вариант B проверяет более крупные touch-зоны на том же размере экрана;
- в Figma включены водительские попапы и окно подтверждения опасного действия.
- дополнительно создана страница `Ночной режим - Горный мастер` для проверки темной темы текущих вариантов A и B.

После визуальной проверки варианта B экран `Комплексы` был поправлен: в карточках экскаваторов с крупными touch-зонами лишние номера самосвалов заменены компактным счетчиком `+N`, чтобы номера не обрезались в узкой карточке.

После уточнения по иконкам техники v7 зафиксировано:

- экскаваторы тоже номерные;
- если экскаватор показан иконкой или плиткой, номер должен быть на самой иконке/плитке;
- самосвал и экскаватор внутри мобильной плитки должны занимать почти всю рабочую площадь;
- номер техники оформляется в стиле приложения: `Manrope Bold`, крупный размер, графитовый цвет `#2E3D48`, светлый контур/ореол `#FAFCF7` около 72% и легкая тень.
- номер техники центрируется по самому квадрату/плитке, а не по силуэту иконки: иконка только обозначает тип техники, а номер является главным идентификатором.
- в рабочих плитках не использовать нижние или мелкие служебные подписи: силуэт показывает тип техники, цвет показывает статус, номер показывает конкретную единицу.
- большие карточки ассетов используют всю площадь квадрата `190 x 184`: PNG и номерный слой центрируются по всей карточке.
- для больших карточек принят номер `font-size 128`, `line-height 90`; для мини-плиток - `font-size 80`, `line-height 64`.

Для модуля Горного мастера также зафиксировано требование дневного и ночного режима:

- дневной режим остается базовым для офиса и обычной работы;
- ночной режим нужен для смены при слабом освещении;
- ночной режим не должен менять смысл статусных цветов;
- в ночном режиме темная основа, приглушенные карточки и светлый текст должны снижать нагрузку на глаза;
- для финальной ночной темы желательно подготовить прозрачные PNG или SVG техники, потому что текущие PNG-ассеты имеют белый фон.

После принятия финального вида плиток техники рабочие экраны Figma обновлены в обеих темах:

- дневные варианты `A - Плотный рабочий` и `B - Крупные touch-зоны`;
- ночные варианты `A - Плотный рабочий / ночь` и `B - Крупные touch-зоны / ночь`;
- старые компактные блоки самосвалов заменены на силуэтные плитки карьерного самосвала с крупным номером по центру;
- экскаваторы также показываются номерными иконками;
- в карточках комплексов добавлена компактная иконка экскаватора с номером;
- статусные цвета плиток синхронизированы с рабочими зонами и сохраняют одинаковый смысл в дневной и ночной теме.

После проверки ночной темы белый фон внутри PNG-иконок техники заменен на темный фон `#0E1822` только на странице `Ночной режим - Горный мастер`. Дневные иконки не менялись.

По дашбордам зафиксировано ограничение текущего Figma-набора: экраны `Итоги смены` и `Итоги - Экс 5` пока являются стартовыми экранами показателей. Полноценные графические дашборды с диаграммами, гистограммами и графиками нужно сделать отдельным следующим экранным набором.

Для экранов `Комплексы` принято правило подсветки состояния комплекса от статуса экскаватора:

- экскаватор является основой комплекса;
- если экскаватор остановлен, вся карточка комплекса подсвечивается мягким красным фоном и красной рамкой;
- если экскаватор в паузе, ОФР или ожидании, вся карточка комплекса подсвечивается мягким желтым фоном и желтой рамкой;
- рабочие комплексы не заливаются зеленым, чтобы не перегружать экран;
- самосвалы внутри комплекса сохраняют собственный статусный цвет.

Для дополнительного офисного режима создана Figma-страница `Десктоп - Горный мастер`:

- desktop не заменяет mobile-first подход, а дополняет его для работы в офисе/АТЦ;
- левую боковую панель не использовать;
- desktop-версия строится по образу рабочей админки: верхняя строка, горизонтальные вкладки, широкая рабочая область, компактная навигация справа;
- собраны 4 экрана `1440 x 900`: рабочая доска комплексов и дашборды в дневной и ночной теме;
- в рабочей доске desktop используется больше пространства для DnD, списка комплексов, поиска, гаража и панели действий с техникой;
- в desktop-дашбордах можно показывать больше графиков одновременно, не пытаясь ужимать их до мобильной плотности.

## 1. Рабочая цель

Модуль Горного мастера - это мобильное рабочее место управления сменой и расстановкой техники.

Основная задача: дать Горному мастеру быстро видеть живую расстановку комплексов, менять состав самосвалов, видеть состояние техники и контролировать факт смены без перегруза диспетчерской отчетностью.

Принципы:

- основной формат - смартфон в вертикальной ориентации;
- desktop в офисе/АТЦ - дополнительный режим;
- планшет возможен позже, но первый UX проектируется под телефон;
- минимум текста и кнопок;
- максимум быстрых действий через понятные зоны и drag-and-drop;
- опасные действия подтверждаются;
- диспетчерская логика, журналы и подробные отчеты не переносятся в мобильный модуль Горного мастера.
- все основные рабочие экраны должны иметь дневную и ночную версию отображения.

## 2. Роли и границы ответственности

### Горный мастер

Может:

- открыть свою смену после закрытия предыдущей смены;
- завершить свою смену через подтверждение;
- видеть текущую живую смену в режиме наблюдателя до открытия своей смены;
- менять расстановку самосвалов между экскаваторами;
- отправлять технику в `ОФР`, `Ремонт`, `Резерв` и возвращать в `Работает` через подтверждение;
- видеть текущие итоги смены;
- видеть параметры работы экскаватора read-only.

Не должен:

- менять горизонт, блок, загрузки, отправки и точки разгрузки экскаватора;
- вести диспетчерский журнал простоев;
- редактировать причины и длительности простоев;
- видеть перегруженные таблицы рейсов на мобильном экране.

### Машинист экскаватора

Может:

- вводить и менять параметры работы экскаватора: горизонт, блок, загрузки, отправки, точки разгрузки;
- выбирать точку разгрузки при отправке рейса;
- видеть назначенные Горным мастером самосвалы;
- видеть ожидаемые самосвалы, которые еще не перешли фактически;
- не получать модальные попапы при перестановках самосвалов.

### Водитель самосвала

Получает попап при смене назначенного экскаватора.

Безопасное переназначение:

- текст: `На Экскаватор 6`;
- кнопка: `ОК`;
- назначение применяется сразу;
- `ОК` только закрывает уведомление.

Рискованное переназначение:

- текст: `На Экскаватор 6`;
- кнопки вертикально: `Перейти сейчас`, `Отложить на 5 минут 04:37`;
- если водитель ничего не нажал, поведение такое же, как `Отложить на 5 минут`;
- если за время отсрочки появился рейс, переход ждет окончания рейса.

## 3. Состояния смены

### До открытия своей смены

Горный мастер видит живую смену предыдущего мастера в режиме наблюдателя.

Доступно:

- `Комплексы` на просмотр;
- `Техника` на просмотр;
- `Итоги` на просмотр;
- плашка `Наблюдение`;
- текст `Ждем закрытия предыдущей смены`.

Недоступно:

- drag-and-drop;
- смена статусов техники;
- начало своей смены, пока предыдущая не закрыта;
- завершение смены.

В режиме наблюдателя не нужно рисовать заблокированные серые кнопки. Действия изменения просто отсутствуют.

### После закрытия предыдущей смены

У нового Горного мастера активируется действие `Начать смену`.

`Начать смену` выполняется только через подтверждение.

### Во время своей смены

Главное действие меняется на `Завершить смену`.

`Завершить смену` выполняется только через подтверждение.

Расстановка и действия с техникой становятся доступны.

Новая смена наследует живую расстановку, потому что производственный процесс безостановочный.

## 4. Навигация модуля

Нижняя навигация принадлежит общему приложению сотрудника, а не производственному модулю.

Внутри модуля Горного мастера используется главный экран с рабочими входами:

- `Смена` - верхний статусный блок и контекстное действие;
- `Комплексы` - главный вход, крупная плитка на всю ширину;
- `Техника` - вторичный вход;
- `Итоги` - вторичный вход.

Главные плитки показывают только короткие счетчики:

- `Комплексы`: экскаваторы в работе, пустые комплексы;
- `Техника`: ремонт, ОФР или другое `Не в работе`;
- `Итоги`: `м3`, рейсы текущей смены.

Плитки не превращаются в отчет.

## 5. Экран `Комплексы`

Назначение: управлять расстановкой самосвалов по экскаваторам.

Первый экран:

- сетка экскаваторов в 2 колонки;
- фиксированный порядок по номеру;
- цветовой статус экскаватора;
- короткий состав самосвалов на плитке;
- если состав не помещается - первые номера и `+N`;
- пустой комплекс показывает метку `Пусто`.

Цветовые состояния экскаваторов:

- зеленый - работает;
- желтый - пауза, ограничение, ОФР;
- красный - ремонт, недоступен;
- серый - резерв или не используется.

Нажатие на экскаватор открывает карточку комплекса выбранного экскаватора.

## 6. Карточка комплекса

Карточка комплекса отвечает только за расстановку.

Показывает:

- выбранный экскаватор;
- закрепленные самосвалы;
- `Гараж` со свободными самосвалами;
- компактный поиск самосвала по номеру;
- зону `Ожидаем`, если есть ожидаемые переходы;
- самосвалы с будущим переходом у старого экскаватора с меткой `-> Экс 6`.

Не показывает:

- настройки экскаватора;
- горизонт и блок как редактируемые параметры;
- загрузки;
- точки разгрузки;
- графики и подробные отчеты.

Механика:

- назначение самосвала применяется сразу;
- зеленой галочки подтверждения комплекса нет;
- снять самосвал можно перетаскиванием из зоны экскаватора в `Гараж`;
- водитель снятого самосвала видит `Ожидайте назначения`;
- система не задает уточняющих вопросов при снятии из комплекса;
- быстрый перенос между экскаваторами должен быть возможен;
- цели других экскаваторов появляются только во время перетаскивания, не постоянно.

Поиск самосвала:

- поле всегда видно сверху;
- вводится чистый номер, например `25`;
- система показывает `Б-25` или `NHL-25`;
- поиск идет по всему парку;
- занятые самосвалы показываются приглушенно с подписью текущего экскаватора;
- при вводе номера полная сетка скрывается, показываются результаты поиска;
- если ничего не найдено - `Не найдено`;
- очистка крестиком внутри поля.

## 7. Переходы самосвалов между экскаваторами

Безопасное окно:

- первые 5 минут после подтвержденной разгрузки;
- если нет активного рейса;
- назначение применяется сразу.

Рискованное окно:

- больше 5 минут после разгрузки без активного рейса;
- водитель мог уже вернуться к старому экскаватору, встать в очередь или начать погрузку;
- водитель получает выбор `Перейти сейчас` или `Отложить на 5 минут`.

Активный рейс:

- переназначение ждет окончания текущего рейса;
- текущий рейс остается за старым экскаватором.

Отложенный переход:

- у старого экскаватора самосвал остается в списке с меткой `-> Экс 6`;
- у нового экскаватора самосвал показывается в зоне `Ожидаем`;
- по самосвалу в зоне `Ожидаем` нельзя создать рейс;
- если старый экскаваторщик создал рейс за время отсрочки, переход переносится на окончание рейса.

## 8. Экран `Техника`

Назначение: менять рабочий статус техники.

Структура:

- вкладки `Экскаваторы` и `Самосвалы`;
- по умолчанию открываются `Экскаваторы`;
- внутри вкладки зоны: `Работает`, `ОФР`, `Ремонт`, `Резерв`;
- `Работает` основная зона;
- остальные зоны компактнее.

Смена статуса:

- через drag-and-drop между зонами;
- переход через границу `Работает` требует подтверждения в обе стороны;
- причины и комментарии не нужны;
- опасным считается действие, которое выводит технику из работы или возвращает ее в работу.

Самосвал в `ОФР`, `Ремонт`, `Резерв`:

- остается в комплексе, если Горный мастер сам его не снял;
- становится неактивным для рейсов.

Экскаватор в `ОФР`, `Ремонт`, `Резерв`:

- самосвалы остаются в комплексе;
- Горный мастер сам решает, снять их или переставить.

## 9. Карточка экскаватора

Карточка экскаватора отвечает за параметры работы экскаватора.

Горный мастер видит:

- номер экскаватора;
- статус;
- горизонт;
- блок;
- загрузки;
- отправки;
- точки разгрузки;
- объем и рейсы текущей смены;
- компактную метку свежести данных, если она не перегружает экран, например `обн. 10:42`.

Горный мастер не меняет:

- горизонт;
- блок;
- загрузки;
- отправки;
- точки разгрузки.

Все это меняет только машинист экскаватора в своем интерфейсе.

Изменения параметров экскаватора у Горного мастера обновляются тихо, без попапов. Если нужно, свежесть данных показывается компактной меткой времени.

## 10. Экран `Итоги`

Назначение: быстро понять факт текущей смены и сравнить экскаваторы.

Основной режим:

- текущая смена.

Дополнительный режим:

- `Сутки` через компактный переключатель сверху;
- суточный режим не основной.

Верхняя сводка:

- `м3`;
- `рейсы`;
- `самосвалы в работе`;
- `Не в работе`.

Не показывать в первом варианте:

- план;
- процент выполнения;
- причины простоев;
- длительности простоев;
- журнал простоев;
- историю рейсов.

Разбивка по экскаваторам:

- горизонтальные строки/полосы;
- `Экс`;
- `м3`;
- `рейсы`;
- `самосвалы`;
- `Не в работе`.

Подробности выбранного экскаватора:

- только текущая смена;
- без списка истории рейсов;
- переключатель `Породы / Разгрузка`;
- одновременно показывается одна разбивка;
- подробные рейсы и история остаются для отдельного отчета или следующего этапа.

## 11. Показатель `Не в работе`

Название `простои/ОФР` не используется в мобильном интерфейсе Горного мастера.

`Не в работе` - это укрупненный сигнал доступности техники/состава.

В модуль Горного мастера не переносится диспетчерская логика простоев:

- причины;
- длительности;
- журналы;
- служебные разборы;
- отчетные таблицы простоев.

Детальный учет простоев остается в диспетчерском, механическом и отчетном контурах.

## 12. Цветовая логика самосвалов

- зеленый - меньше 5 минут после разгрузки, лучший момент для переназначения;
- желтый - больше 5 минут после разгрузки без активного рейса, риск очереди/погрузки;
- синий - в рейсе;
- бирюзовый/фиолетовый - переход запланирован;
- красный - ремонт/недоступен;
- серый - свободен без экскаватора в `Гараже`.

Типы самосвалов:

- `Б` - БелАЗ;
- `NHL` - NHL;
- обозначения: `Б-25`, `NHL-64`;
- поиск по чистому номеру без префикса.

Цвет типа и цвет статуса не смешивать:

- тип показывать текстом/иконкой;
- статус показывать контуром, точкой или цветовой меткой.

## 13. Что не входит в первый Figma-вариант

Не включать:

- диспетчерский журнал простоев;
- редактирование горизонтов, блоков и точек разгрузки Горным мастером;
- список истории рейсов в мобильных подробностях экскаватора;
- план и процент выполнения;
- сложные таблицы отчетов;
- подтверждения на каждое быстрое назначение самосвала;
- постоянное отображение всех экскаваторов как целей переноса на маленьком экране.

Можно оставить на следующий этап:

- отдельный отчет истории рейсов;
- полноценный механизм сменного плана;
- расширенная аналитика по простою;
- desktop-версия с более плотными таблицами;
- планшетная компоновка.

## 14. Figma-экраны для первого набора

Минимальный набор фреймов:

1. Главный экран: активная смена.
2. Главный экран: режим наблюдателя.
3. Экран `Комплексы`: сетка экскаваторов.
4. Карточка комплекса: расстановка, `Гараж`, поиск, DnD.
5. Карточка комплекса: ожидаемый переход и метки `-> Экс`.
6. Экран `Техника`: вкладка `Экскаваторы`.
7. Экран `Техника`: вкладка `Самосвалы`.
8. Карточка экскаватора: read-only параметры работы.
9. Экран `Итоги`: сменная сводка и строки экскаваторов.
10. `Итоги`: подробности экскаватора, переключатель `Породы / Разгрузка`.
11. Водительский попап безопасного переназначения.
12. Водительский попап рискованного переназначения.
13. Подтверждение опасного действия с техникой.

Для первого Figma-подхода лучше сделать 2 визуальных варианта:

- вариант A: максимально плотный рабочий интерфейс;
- вариант B: более крупные touch-зоны для проверки удобства на телефоне.

После сравнения выбрать базу для будущего кода.

## 15. Иконки мобильного интерфейса

По результату просмотра первого Figma-варианта решено отдельно согласовать набор иконок до замены элементов в рабочих экранах.

Причина: в мобильном интерфейсе карточки самосвалов не должны восприниматься как прямоугольные карточки или банковские карты. Самосвал должен читаться как техника даже в маленьком размере, а нижняя навигация должна выглядеть привычно для мобильного приложения: иконка сверху, короткая подпись снизу.

В Figma-файле `Горный мастер - mobile UX concept` создана отдельная страница `Иконки v1` для согласования:

- варианты иконки самосвала;
- варианты иконки экскаватора;
- нижняя навигация общего приложения;
- внутренние входы производственного модуля;
- пример плитки самосвала с крупным номером;
- мини-иконки 24 px для проверки читаемости.

Предварительная рекомендация:

- для самосвалов использовать вариант `Самосвал C`: крупный номер сохраняется, но форма плитки читается как техника;
- нижнее меню делать как иконку с короткой подписью;
- статусы техники оставить цветом, не превращая каждый статус в отдельную пиктограмму;
- опасные действия не прятать только за иконкой, оставлять подтверждение текстом.

До утверждения иконок существующие рабочие экраны Figma не менять. После утверждения сначала заменить иконки на одном экране `Комплексы`, проверить читаемость, затем переносить на остальные экраны.

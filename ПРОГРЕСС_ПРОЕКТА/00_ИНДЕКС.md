# Индекс прогресса проекта

Папка создана для хранения текущего состояния проекта, принятых решений, планов, версий документов и краткой памяти для продолжения работы без зависимости от окна чата.

## Главные файлы

- `01_ТЕКУЩЕЕ_СОСТОЯНИЕ.md` - краткая картина проекта на текущий момент.
- `02_ЖУРНАЛ_РЕШЕНИЙ.md` - важные архитектурные и организационные решения.
- `03_КАК_ПРОДОЛЖАТЬ_РАБОТУ.md` - правила, как продолжать проект после перерыва или переполнения памяти.
- `04_БЛИЖАЙШИЕ_ЗАДАЧИ.md` - список следующих рабочих шагов.
- `08_ЭТАПЫ_РАБОТЫ.md` - этапы работы над проектом и порядок запуска MVP.
- `09_ТЗ_MVP_ПРОИЗВОДСТВЕННОГО_КОНТУРА.md` - рабочее ТЗ первого производственного MVP.
- `10_МОДЕЛЬ_ДАННЫХ_MVP.md` - первая модель данных ядра MVP производственного контура.
- `11_СПРАВОЧНИКИ_MVP.md` - найденные и выгруженные справочники для MVP.
- `11_ДАННЫЕ_СПРАВОЧНИКОВ_MVP\` - CSV-выгрузки справочных данных из найденных Excel-файлов.
- `12_АДМИНКА_И_ПЕРВИЧНАЯ_РЕГИСТРАЦИЯ.md` - сценарий создания водителя, выдачи доступа, первичной регистрации и контроля конфликтов.
- `13_ДОРОЖНАЯ_КАРТА_MVP.md` - этапы до первого рабочего MVP и формат отображения прогресса.
- `14_ПРАВИЛА_АДМИНКИ_И_СПРАВОЧНИКОВ.md` - правила ведения справочников и минимальный состав админки MVP.
- `15_ПЕРВЫЙ_ТЕХНИЧЕСКИЙ_КАРКАС_MVP.md` - первый технический каркас MVP: ядро, модули, интерфейсы, таблицы, порядок разработки и критерии готовности.
- `16_ОТЧЕТЫ_ПЕРВОЙ_ОЧЕРЕДИ_MVP.md` - отчеты первой очереди: объемы за смену, объемы за сутки и суточный отчет заказчику к 08:00.
- `17_ЭКРАНЫ_MVP.md` - первый набор экранов MVP: единый вход, водитель, машинист экскаватора, горный мастер, диспетчер, админка, отчеты и витрина руководства.
- `18_ПЛАН_ЭТАПА_2_ТЕХНИЧЕСКИЙ_КАРКАС_ЯДРА.md` - план перехода к созданию технического каркаса ядра: структура проекта, модули, модели, загрузка справочников и критерии готовности.
- `19_ТЕХНИЧЕСКИЙ_СТАТУС_КАРКАСА.md` - фактический статус созданного Django-каркаса, моделей, миграций, админки и загруженных справочников.
- `20_ДЕМОНСТРАЦИОННЫЙ_СЦЕНАРИЙ_MVP.md` - первый сценарий показа MVP производственного контура по ролям.
- `21_СТАРТ_ПОСЛЕ_ЛИМИТА.md` - точка продолжения после обновления лимита.
- `22_СУТОЧНЫЙ_ОТЧЕТ_ЗАКАЗЧИКУ.md` - разбор Excel-образца суточного отчета заказчику и первая структура веб-отчета.
- `23_ЧЕКЛИСТ_ПИЛОТНОЙ_ПРОВЕРКИ_ОТЧЕТОВ.md` - контрольный список проверки отчетов, витрины и Excel-выгрузок перед переходом к пилотному запуску.
- `24_РЕЕСТР_ТЕКУЩИХ_EXCEL_ОТЧЕТОВ.md` - реестр текущих Excel-отчетов диспетчерской и матрица покрытия этих форм текущим MVP.
- `25_СВЕРКА_СУТОЧНОГО_ОТЧЕТА_ЗАКАЗЧИКУ.md` - подробная сверка действующей формы `Отчет_Коппер. Рисорсез_Март.xlsx` с текущим суточным отчетом MVP.
- `26_СВЕРКА_ПОЧАСОВОГО_ОТЧЕТА.md` - разбор формы `почасовой Март.xlsx` и решение по первой MVP-замене через группировку рейсов по часу выполнения.
- `27_СВЕРКА_ОР_ККД_СКДР.md` - разбор формы `ОР ККД СКДР март.xlsx` и решение по MVP-учету ожидания разгрузки через события простоев самосвалов.
- `28_СВЕРКА_СПРАВОЧНИКА_ПОРОД_И_ПЛОТНОСТЕЙ.md` - сверка файла `удельный_веса_руд_и_пород_Малмыжского_местородения.xlsx` с текущим справочником пород, плотностей и коэффициентов разрыхления MVP.
- `29_ГРАНИЦЫ_ПЕРВОГО_ПИЛОТА.md` - что входит в первый пилот производственного MVP, что проверяется на пилоте и что осознанно переносится после пилота.
- `30_СЦЕНАРИЙ_ПИЛОТНОГО_ЗАПУСКА.md` - короткий пошаговый сценарий пилотной проверки: подготовка базы, роли, действия, отчеты, сверка и критерии результата.
- `31_ЖУРНАЛ_ЗАМЕЧАНИЙ_ПИЛОТА.md` - шаблон для фиксации замечаний пилота, решений по ним, приоритета и распределения на до запуска / после запуска / отдельный модуль.
- `32_СТРУКТУРА_АДМИНКИ_MVP.md` - структура админки MVP: экраны, права, доступы, справочники, модель данных, проверки, выгрузки и критерии готовности.

## Основные документы проекта

- `..\ARCHITECTURE_DRAFT.md` - черновик архитектуры всей учетной системы.
- `..\ПЛАН_ПРОИЗВОДСТВЕННОГО_КОНТУРА.md` - первый план производственного контура.
- `..\outputs\Презентация_для_запуска_учетной_системы (эталон).pptx` - эталонная презентация для руководства, если файл присутствует.
- `..\outputs\Презентация_для_запуска_учетной_системы.pptx` - текущая рабочая презентация.

## Как пользоваться

Перед началом новой сессии нужно открыть этот индекс и файл `01_ТЕКУЩЕЕ_СОСТОЯНИЕ.md`.

После важного обсуждения или изменения нужно обновить:

1. `01_ТЕКУЩЕЕ_СОСТОЯНИЕ.md`, если изменилась общая картина.
2. `02_ЖУРНАЛ_РЕШЕНИЙ.md`, если принято важное решение.
3. `04_БЛИЖАЙШИЕ_ЗАДАЧИ.md`, если поменялся план действий.

Текущая организационная логика: проект ведется в одной общей папке, а этапы оформляются отдельными документами внутри папки прогресса.

# Журнал замечаний пилота

Дата создания: 19.06.2026

## Назначение

Документ нужен для фиксации всех замечаний во время демонстрационного или реального пилота.

Главное правило: замечание должно превращаться в конкретное решение, а не оставаться устной фразой.

Рабочая веб-страница журнала: `/reports/pilot-feedback/`.

Excel-выгрузка журнала: `/reports/pilot-feedback/export/`.

## Как заполнять

Для каждого замечания фиксировать:

- дату;
- роль или отдел;
- экран или отчет;
- что произошло;
- насколько это критично;
- что нужно сделать;
- куда относится доработка;
- статус решения.

## Приоритеты

Использовать четыре уровня:

| Приоритет | Значение | Когда ставить |
|---|---|---|
| P0 | Блокирует пилот или запуск | Нельзя пройти основной цикл, данные теряются, отчет не формируется |
| P1 | Нужно до реального запуска | Работать можно, но риск ошибок высокий |
| P2 | Можно после запуска | Не мешает пилоту, но улучшает удобство или полноту |
| P3 | Идея развития | Новый большой функционал или улучшение на будущее |

## Категории

| Категория | Что включает |
|---|---|
| Интерфейс | лишние клики, непонятные кнопки, неудобный экран |
| Данные | не хватает поля, неправильное значение, спорный справочник |
| Рейс | создание, завершение, переходящий рейс, объем, тоннаж |
| Смена | открытие, закрытие, пересменка, показатели техники |
| Назначение | расстановка, подтверждение, переназначение |
| Простои | механика, ожидание разгрузки, длительность, причина |
| Отчет | экран отчета, фильтры, группировки, Excel |
| Витрина | показатели руководства, графики, динамика |
| Права | доступы, роли, кто что может делать |
| Следующий модуль | КИП/КТГ, КИО/КТГ, ТО, перегоны, мобильное приложение |

## Статусы

| Статус | Значение |
|---|---|
| новое | замечание записано, решение еще не принято |
| принято в работу | нужно исправить |
| вопрос к пользователю | нужно уточнение у отдела или руководства |
| отложено после запуска | не блокирует запуск |
| отдельный модуль | слишком большой блок для текущего MVP |
| сделано | исправлено и проверено |
| отклонено | принято решение не делать |

## Вопросы, которые нужно закрыть на пилоте

1. Достаточно ли группировки по часу вместо старой почасовой матрицы.
2. Кто фиксирует ожидание разгрузки ККД/СКДР.
3. Как считать среднее `мин на 1 а/с`.
4. Как связать укрупненные грузы с точными породами заказчика.
5. Нужна ли плотность по карьеру.
6. Какие Excel-отчеты диспетчерская обязана сдавать каждый день.
7. Какие действия пользователи считают лишними.
8. Какие данные невозможно вводить стабильно в карьере.
9. Какие показатели руководство хочет видеть первыми.

## Обновление 20.06.2026: зафиксирован актуальный порядок запуска сервера после ремонта среды Codex

После обновления Codex и ремонта среды зафиксирован новый рабочий порядок запуска локального сервера MVP.

Что было выявлено:

- в окружении процесса Codex одновременно появлялись `PATH` и `Path`;
- из-за этого PowerShell/.NET мог падать с ошибкой `Элемент с тем же ключом уже был добавлен`;
- `Get-ChildItem Env:` мог ломаться;
- скрытый запуск сервера через `Start-Process` работал нестабильно;
- старая кодировка PowerShell `chcp 866` и `$OutputEncoding = us-ascii` могла приводить к кракозябрам в русских файлах.

Что исправлено в среде:

- в постоянных переменных Windows проверено отсутствие дубля, остается нормальный `Path`;
- создан бэкап окружения `ENV_BACKUP_2026-06-20`;
- создан профиль PowerShell `C:\Users\swwba\Documents\WindowsPowerShell\Microsoft.PowerShell_profile.ps1`;
- для текущего пользователя включен `ExecutionPolicy RemoteSigned`;
- профиль PowerShell при старте убирает process-level дубль `PATH/Path`, оставляет один `Path` и включает UTF-8.

Актуальное правило запуска:

- встроенный терминал Codex для ручной работы лучше держать на `Command Prompt`;
- PowerShell не использовать для ручного запуска сервера без необходимости;
- запускать сервер стандартно через `START_SERVER_MVP.bat`;
- останавливать через `STOP_SERVER_MVP.bat`;
- не использовать ручной запуск через глобальные `python` или `py`;
- не использовать ручные команды `python manage.py runserver`, `py manage.py runserver` и запуск `.ps1` без необходимости;
- если есть сомнения по зависшим процессам: сначала `STOP_SERVER_MVP.bat`, затем `START_SERVER_MVP.bat`.

Проверка:

- `127.0.0.1:8000` отвечает `200`;
- после смены встроенного терминала на `Command Prompt` сервер поднялся штатно;
